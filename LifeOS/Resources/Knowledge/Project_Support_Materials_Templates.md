## Template Usage Guidelines

### When to Use Each Template

1. **Project Reference Materials Index**: Use when starting a new project or organizing existing project materials
2. **Research Material Filing Template**: Use for each research item or reference material added to a project
3. **Technical Specification Reference**: Use for technical documentation, requirements, or specifications
4. **Resource Collection Template**: Use when organizing tools, resources, or collections needed for a project
5. **Cross-Reference Tracking Template**: Use to maintain links and tracking throughout the project lifecycle

### Integration with GTD System

- All templates include GTD classification fields
- Action items are clearly marked with context assignments
- Processing logs track movement through the system
- Cross-references maintain system integrity

### Maintenance Schedule

- **Weekly**: Update active project references and resource links
- **Monthly**: Review and validate all cross-references
- **Project Completion**: Archive materials and update master indexes
- **Quarterly**: Clean up outdated references and consolidate resources

### Best Practices

1. **Consistent Naming**: Use standardized naming conventions for all materials
2. **Regular Reviews**: Schedule regular reviews to maintain system integrity
3. **Context Assignment**: Always assign appropriate contexts to action items
4. **Link Validation**: Periodically validate all cross-references
5. **Archive Management**: Properly archive completed project materials
